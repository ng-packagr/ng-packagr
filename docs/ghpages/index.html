When Not To

  • You have existing, customized build scripts — does it break?
  • Third-party dependencies can be tricky (#129)
  • Angular 5 works â€¦ maybe?

When To

  • Less Configuration, More Convention
  • Small and focused: just an "orchestrator" for a complex build tool pipeline
  • Secondary entry points (like @angular/cdk/portal)
  • Complementary to Angular CLI, Nx Workspace, or whatsoever!
